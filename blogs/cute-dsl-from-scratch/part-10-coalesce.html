The Art of Coalescing

Understand how complex, hierarchical memory layouts collapse into simple flat forms. Master the principle behind one of GPU optimization's most powerful techniques.

April 2026 ~15 minute read Part 10 of 13
§10 — Intuition

The Problem: Nested Complexity

You've been reading about GPU memory layouts. You understand that a shape describes dimensions, and a stride describes how to skip between elements. But then you encounter this: (2, (1, 6)) : (1, (6, 2)). A nested tuple. A hierarchical layout.

Your question: "What does this actually do?"

The answer: despite the nesting, despite the hierarchy, this layout accesses exactly 12 memory addresses in a perfectly linear stride-1 pattern. The nesting is just packaging. The real access pattern is simple.

Core Realization

Many complex memory layouts are functionally equivalent to simple flat layouts. Coalescing is the operation that finds this equivalence and collapses the hierarchy.

Think of it like fractions. 12/4 and 3/1 are different representations of the same value. Coalescing is the simplification step.

Why This Matters

  • Performance: Flat layouts compile to efficient code. Complex ones require index arithmetic.
  • Reasoning: It's easier to reason about a flat layout than a deeply nested one.
  • Composition: When combining layouts, knowing the coalesced form helps predict the result.

The Pattern to Remember

If consecutive modes have no gaps or overlaps—if stride[i+1] = size[i] × stride[i]—they merge. Otherwise, they stay separate.

§10 — Definition

The Formal Definition

Coalesced Layout

A layout R is a coalesced layout of layout A if:

  1. Same size: |R| = |A|
  2. Flat shape: depth(R) ≤ 1 (no nesting)
  3. Same function: R(c) = A(c) for all coordinates c

What does this mean?

  • Same size: Both layouts address the same number of memory elements. A 12-element layout coalesces to a 12-element layout.
  • Flat shape: The result has no nesting. It's a scalar, or a simple tuple. Not a tuple of tuples.
  • Same function: The memory addresses accessed are identical. The meaning of the layout is preserved.

In essence: a layout is fundamentally a function from coordinates to memory addresses. Coalescing preserves the function while simplifying the representation.

§10 — Contiguity

Contiguity: When Do Modes Merge?

Coalescing works by merging adjacent modes. But not all pairs can merge. The question is: when are two modes contiguous?

Contiguity Condition:
stride[i+1] = size[i] × stride[i]

If this holds, modes i and i+1 are contiguous. They access memory without gaps or overlaps, so they can safely merge.

Why Contiguity Matters

  • Row-major: Adjacent dimensions are naturally contiguous. You can coalesce the entire layout.
  • Column-major: Also contiguous. You can coalesce the entire layout.
  • Custom strides: Gaps appear quickly. Only contiguous pairs merge.

§10 — Examples

Real-World Examples

Example 1: Column-Major (Contiguous)

Shape (3, 4, 2), stride (1, 3, 12). All modes are contiguous:

  • Mode 0→1: 3 = 3×1 ✓
  • Mode 1→2: 12 = 4×3 ✓

Result: 24 : 1 (perfectly flat, one long linear access)

Example 2: Partial Coalescing

Shape (2, 3, 5, 4), stride (1, 2, 10, 50):

  • Mode 0→1: 2 = 2×1 ✓ Merge → 6
  • Mode 1→2: 10 ≠ 3×2 ✗ Cannot merge (gap)
  • Mode 2→3: 50 = 5×10 ✓ Merge → 20

Result: (6, 20) : (1, 10) (partially coalesced)

Example 3: Hierarchical to Flat

The example from the start: (2, (1, 6)) : (1, (6, 2))

When flattened and coalesced over the full integral coordinate space, this maps linearly: every index 0-11 maps to address 0-11.

Result: 12 : 1 (completely flat)

§10 — Summary

Why This Matters

Coalescing is a fundamental tool in GPU optimization because:

  • Performance: Flat layouts compile to efficient load/store sequences. Complex ones require index arithmetic at runtime.
  • Reasoning: Understanding what a complex layout coalesces to tells you if access is efficient or scattered.
  • Composition: When you compose multiple layouts (common in CUTLASS), knowing how they coalesce helps predict the final result.
  • Debugging: If a kernel is slow, check: does its layout coalesce? If not, you've found a bottleneck.
Key Insight

Coalescing is simplification. A coalesced layout is the minimal flat representation of what might be a deeply nested memory access pattern. Master it, and you understand one of the most powerful tools in GPU programming.
