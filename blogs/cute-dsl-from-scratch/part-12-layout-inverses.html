Layout Inverses

Layouts map coordinates to offsets. Their inverses map offsets back to coordinates — powering vectorized copies and instruction admissibility checks.

April 2026~15 minute readPart 12 of 13
§12 — Intuition

Going the other direction

A layout L answers one question: "given a logical coordinate, what is the memory offset?" The inverse answers the opposite: "given a memory offset, which logical coordinate produced it?"

Think of a simple column-major matrix (4, 8) : (1, 4). It maps coordinate (row, col) → row × 1 + col × 4. If I show you offset 9, can you find the coordinate? Yes: 9 = 1×1 + 2×4, so coordinate (1, 2). The layout inverse is the function that does this lookup — efficiently, as a layout itself.

This matters in practice for two reasons:

Vectorization

To copy two tensors fast, find the largest contiguous block of offsets they share. The right-inverse tells you this.

Admissibility

A hardware instruction accesses a fixed set of offsets. Does every offset it touches exist in the data layout? The left-inverse answers this.

Crucially, the inverse is not just a lookup table — it is itself a layout, with shape and stride. That means it composes, inverts, and tiles just like any other layout.

Definition

The right-inverse L‡

The right-inverse L‡ of a layout L satisfies: applying L after L‡ gives back the identity — at least on the domain of L‡.

Definition — right-inverse

L‡ is an injective layout such that for all k in its domain:

L( L‡(k) ) = k

That is: L‡ maps an offset k to a coordinate, and L maps that coordinate back to k. The round-trip succeeds.

For a bijective layout (one that covers all of ℤ|L| exactly once), L‡ has size equal to |L| and is a true inverse. For a non-contiguous layout, only a prefix of offsets can be recovered — the right-inverse is smaller.

Non-contiguous layouts

For (4,8):(1,5) the offsets are 0,1,2,3,5,6,7,8,10,… — there's a gap at offset 4. Only the first 4 offsets (0–3) form a contiguous prefix, so the right-inverse has size 4. Starting from offset 5, there is no layout-consistent way to recover the coordinate from just the offset number.

⚡

Key insight: The size of the right-inverse is the length of the maximal contiguous prefix of the image. For fully bijective layouts this equals the full size; for gapped layouts it can be much smaller.

Special case

Compact layouts and full inverses

A layout is compact (bijective on ℤ|L|) when it maps every integer in 0…|L|-1 to a distinct offset covering exactly those offsets. For such layouts, the right-inverse is a true two-sided inverse:

L( L⁻¹(k) ) = k for all k in Z_{|L|}
L⁻¹( L(k) ) = k for all k in Z_{|L|}

Column-major (4,8):(1,4) → right-inverse is 32:1 (the identity, since col-major is already contiguous). Row-major (4,8):(8,1) → right-inverse is (8,4):(4,1), which transposes the layout.

Application 1

Finding the maximum vectorizable block

When copying tensor A to tensor B, we want to move as many elements as possible in a single instruction. A SIMD or memory instruction copies a contiguous block of K offsets at once — so we need to find the largest K such that both A and B agree on their first K elements' physical addresses.

The algorithm uses the right-inverse:

Compute A ∘ B‡ (compose A with right-inverse of B)
Find the largest K where (A ∘ B‡)(k) = k for k = 0,1,…,K-1

If (A ∘ B‡) acts as the identity on the first K inputs,
then those K elements have identical physical layout in both A and B.

Equivalently, you can compute B ∘ A‡ and find the same K. The two formulations give the same answer.

In practice, CuTe combines this algebraic result with hardware knowledge (pointer alignment, SIMD width) to choose the actual vector width. The math gives the upper bound; hardware constrains the floor.

Definition

The left-inverse L†

The left-inverse L† goes the other direction: it maps an offset back to the coordinate that produced it.

Definition — left-inverse

L† is a layout that for all k in the domain of L:

L†( L(k) ) = k

Apply L first (coordinate → offset), then L† (offset → coordinate) and you recover the original coordinate. The composition L† ∘ L is the identity on L's domain.

For injective layouts the left-inverse is unique on the image. For non-injective layouts (those with repeated offsets), L† can take arbitrary values at offsets outside the image — the definition only requires it behaves correctly on offsets that are in the image.

Left vs right — when do they coincide?

For bijective (compact) layouts, the left- and right-inverses are identical — they're both the true inverse L⁻¹. For other layouts:

Right-inverse L‡

Satisfies L(L‡(k)) = k. Size ≤ |L|. Measures how many contiguous offsets from 0 can be round-tripped. Smaller for gapped layouts.

Left-inverse L†

Satisfies L†(L(k)) = k. Has domain = cosize of L. Larger for gapped layouts (needs to cover all offsets in the image).

Application 2

Admissibility checking

A hardware instruction has a fixed access pattern — it always touches the same set of relative offsets. Before using it, we need to verify it's admissible: every offset it touches must exist in the data layout.

Formally: data layout A : ℤ₍ₐ₎ → ℤ and instruction layout T : ℤ₍ₜ₎ → ℤ. T is admissible for A if:

∀ i ∈ ℤ_{|T|} : A( A†(T(i)) ) = T(i)

Use L† to map the offset T(i) back to a coordinate,
then use L to go forward again. If you get T(i) back,
the offset was genuinely in the image of A.

If the round-trip fails, the offset T(i) is not in A's image — the instruction is trying to access memory that the data layout doesn't reach.
